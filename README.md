# Telegram Shop Bot

Telegram-бот для продажи цифровых подписок и AI-сервисов: каталог товаров, баланс, история транзакций, автозаказы, реферальная система, отзывы и админ-панель.

Проект оформлен как портфолио-кейс. Если нужен бот под заказ или доработка текущего проекта, писать в Telegram: [@yaroslav_blog1](https://t.me/yaroslav_blog1)

## Что умеет бот

- каталог товаров по категориям
- покупка цифровых товаров и ключей
- внутренний баланс пользователя
- история транзакций и заказов
- отзывы и рейтинг товаров
- реферальная система
- промокоды
- бан / разбан пользователей
- админ-панель для управления товарами, ключами и балансом

## Стек

- Python 3.12+
- aiogram 3
- SQLite

## Переменные окружения

| Переменная | Описание |
| --- | --- |
| `BOT_TOKEN` | токен Telegram-бота от BotFather |
| `ADMIN_ID` | Telegram ID администратора |
| `YUKASSA_SHOP_ID` | ID магазина ЮKassa |
| `YUKASSA_SECRET_KEY` | секретный ключ ЮKassa |
| `MIN_DEPOSIT` | минимальная сумма пополнения |
| `CURRENCY` | валюта |
| `MANAGER_USERNAME` | Telegram менеджера / контакта |
| `CHANNEL_USERNAME` | канал для проверки подписки, если используется |
| `REFERRAL_BONUS` | бонус рефереру |
| `REFERRAL_BONUS_NEW` | бонус новому пользователю |


## Как залить проект на GitHub

### 1. Подготовь проект

Проверь, что в проекте нет секретов:

- в `config.py` нет реальных токенов
- файл `.env` не попадает в коммит
- `data/shop.db` не попадает в коммит
- `.gitignore` лежит в корне проекта

Если токен уже был где-то сохранён или засвечен, обязательно перевыпусти его через BotFather.

### 2. Создай репозиторий на GitHub

1. Зайди на GitHub
2. Нажми `New repository`
3. Укажи имя, например `telegram-shop-bot`
4. Нажми `Create repository`

### 3. Инициализируй git локально

В корне проекта выполни:

```bash
git init
git branch -M main
git status
```

Перед коммитом посмотри список файлов. Там не должно быть `.env`, `data/shop.db`, `__pycache__`.

### 4. Сделай первый коммит

```bash
git add .
git commit -m "Initial commit"
```

### 5. Подключи GitHub-репозиторий

Подставь свою ссылку вместо `YOUR_USERNAME`:

```bash
git remote add origin https://github.com/YOUR_USERNAME/telegram-shop-bot.git
git push -u origin main
```

### 6. Если секреты уже случайно попали в git

Если ты уже добавил `.env` или `data/shop.db` в индекс, убери их из отслеживания:

```bash
git rm --cached .env
git rm --cached data/shop.db
git rm --cached -r __pycache__
git commit -m "Remove local secrets and cache files"
git push
```

### 7. Что показать в портфолио

Рекомендую оставить в репозитории:

- `README.md`
- чистый исходный код
- безопасный `.env.example`
- скриншоты бота в папке `assets/`, если захочешь добавить позже

Не выкладывай:

- реальные токены
- платежные ключи
- базу с пользовательскими данными
- приватные переписки и внутренние логи
